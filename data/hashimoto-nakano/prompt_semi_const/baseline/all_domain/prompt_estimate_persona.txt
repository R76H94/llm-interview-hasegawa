# タスク
あなたは対話履歴と既存のスロットを分析し、インタビュー対象者が現時点でどのようなペルソナ属性を持っていると推定できるかを判断します。

# 前提
インタビューで聞き出したインタビュー対象者のペルソナ情報はスロットに格納されています。
ペルソナ属性とは、インタビュー対象者が持つ属性カテゴリ（例：「キャリア・職場」「悩みやストレス」「趣味」「家族」など）のことです。
推定ペルソナとは、インタビューの対話履歴と既存のスロットをもとに、「現時点でこのインタビュー対象者はどのようなペルソナ属性を持っている（または持っている可能性が高い）」を推定した結果です。

# 指示
1. 対話履歴と既存のスロットを分析し、値が埋まっているスロットを特定します。
2. 値が埋まっているスロットのうち、意味のある情報が入っているスロットをもとに、インタビュー対象者のペルソナ属性を推定してください。
  - 意味のある情報が入っているスロットとは、値がnullでなく、かつインタビュー対象者のペルソナ属性を特定するのに役立つ情報が入っているスロットです。
  - 例えば、「現在のキャリア」「リーダー経験」というスロットに現在の勤務先やリーダー経験の有無が入っている場合、これらは「キャリア・職場」というペルソナ属性にまとめることができます。
3. 既存の推定ペルソナを参照し、今回情報が得られていないペルソナ属性は「未」として維持してください。
4. 属性は必要最小限の数に抑えてください。
5. 属性ごとに状態を付与してください。
  - 済：対象者の発話またはスロットからそのペルソナ属性に関連する情報が得られている
  - 未：まだ情報は得られていないが、インタビュー対象者が潜在的に持っていると考えられるペルソナ属性
6. インタビューの結果、新たに明らかになったペルソナ属性があれば、済として推定ペルソナに追加してください。また、推定ペルソナに冗長な属性が複数ある場合は、まとめてください。

# スロットの説明
スロットはインタビュー対象者のペルソナ情報を整理・記録するために使用します。
各スロットはスロット名、値の2つの要素で構成されます。
- スロット名: どのような情報が記録されているかを示すスロットの名前です。
- 値: インタビューで話された情報を端的にまとめて記録します。nullの場合は、まだ情報が聞き出されていない状態です。
- 例: {{"現在のキャリア": "大学病院の内科で勤務", "ストレス解消方法": null}}

# 出力指示
コメントや説明は不要です。
推定したペルソナ属性名と状態（未/済）を1行ずつ出力してください。
ペルソナ属性名は抽象的なキーワードかつ重複しないようにしてください。
インタビューの対話履歴を見て、既に情報が得られたペルソナ属性は、その属性の後に「済」と記載してください。
インタビューでまだ情報が得られていないペルソナ属性には「未」と記載してください。

例えば、以下のように考えて出力してください.
 1. "現在のキャリア"、"リーダー経験"、"悩みや不満点"、"職場の雰囲気"、"ストレス解消方法"というスロットの値に情報が入っていることを確認する。
 2. 情報が入っているスロットを抽象的なキーワードでまとめる。"現在のキャリア"、"リーダー経験"、"職場の雰囲気"は"キャリア・職場"としてまとめる。"悩みや不満点"、"ストレス解消方法"は"悩みやストレス"としてまとめる。
 3. まとめたキーワードを推定ペルソナとして出力する。
# 例
キャリア・職場: 済
悩みやストレス: 未

[推定ペルソナ属性名]: 未
[推定ペルソナ属性名]: 済
