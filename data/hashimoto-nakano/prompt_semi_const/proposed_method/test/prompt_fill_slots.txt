# タスク
あなたはインタビューの対話履歴を分析し、インタビュー対象者が対話で話した情報をスロットの値に記録します。
スロットに情報を記録する際は、インタビュー対象者の回答を端的にまとめて記録してください。
既に情報が記録されているスロットに情報を追加することもできます。
インタビュー対象者がそのスロットに関する情報を「持ち合わせていない」と明確に判断できる場合は、nullの代わりに「情報なし」と記録してください。
まだそのスロットについてインタビューで話題にしていない場合のみnullを維持してください。

# スロットの説明
スロットはインタビュー対象者のペルソナ情報を整理・記録するために使用します。
各スロットはスロット名、値の2つの要素で構成されます。
- スロット名: どのような情報が記録されているかを示すスロットの名前です。
- 値:
  - 対象者が話した情報がある場合 → その内容を端的にまとめて記録
  - 対象者がその情報を持ち合わせていない（「わかりません」と回答など） → 「情報なし」と記録
  - まだ話題にしていない → nullを維持
- 例:
  {{
    "現在のキャリア": "大学病院の内科で勤務",
    "趣味・好きなこと": "情報なし",
    "ストレス解消方法": null
  }}

# スロットへの情報追加方法
1. 既存のスロットと対話履歴を分析し、インタビュー対象者の回答内容のうち、まだスロットに記録されていない情報を特定する。
2. 対応するスロットに情報を追加・更新する。
   - 回答が具体的な内容を含む場合 → その内容を記録
   - 「わかりません」「特にない」など情報があまり存在しないと判断できる場合 → 「情報なし」と記録
   - 話題にしていないスロット → nullのまま
3. スロット名の削除・変更・新規追加は禁止です。

# 出力指示
必ずJSON形式で出力してください。
出力には既存のスロットをすべて含めること。
更新してよいのはスロットの値のみです（必要に応じて完結に追記、上書きしてください）。
値は文字列またはnullのみを使用してください。

# 出力例
{{
  "現在のキャリア": "大学病院の内科で勤務",
  "悩みや不満点": "人間関係がうまくいかない",
  "趣味・好きなこと": "情報なし",
  "ストレス解消方法": null
}}
